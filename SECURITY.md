# Security Policy

## Supported versions

| Branch | Supported |
|--------|-----------|
| `main` | ✅ Yes    |
| Other  | ❌ No     |

## Reporting a vulnerability

If you discover a security vulnerability, please **do not open a
public GitHub issue**.

Report it exclusively via GitHub Security Advisory:  
👉 [Open a private security advisory](https://github.com/crisis1er/squid-tumbleweed-config/security/advisories/new)

Please include:
- File(s) concerned
- Description of the vulnerability
- Potential impact
- Steps to reproduce if applicable

I will acknowledge the report within **72 hours** and aim to publish
a fix or mitigation within **14 days** depending on severity.

## Scope

Tested on **openSUSE Tumbleweed** and **openSUSE Leap**.  
Compatible with openSUSE-based distributions — minor adaptations
may be required depending on your environment.

Always review and adapt before deploying in any production or
enterprise environment.
